# 输入法、拉丁化、编码与字体总览

这一页整理跨语言学习时最容易混淆的四件事：怎么输入、怎么转写、字符如何编码、字体如何显示。

## 基本原则

- **输入法**解决“怎么打字”：键盘布局、组合键、候选词、音码或形码。
- **拉丁化**解决“怎么用拉丁字母标音”：适合初学和检索，但不能替代原文字。
- **编码**解决“字符是什么”：现代资料优先使用 Unicode / UTF-8。
- **字体**解决“字符长什么样”：同一 Unicode 字符在不同地区可能有不同字形。

## 常见文字系统

| 类型 | 语言例子 | 输入重点 | 学习重点 |
| --- | --- | --- | --- |
| 拉丁字母 | 英语、法语、西班牙语、越南语 | 重音、附加符号、死键 | 区分字母本身和音标 |
| 西里尔字母 | 俄语、乌克兰语等 | 键盘布局或音译输入 | 字母形状和拉丁字母假朋友 |
| 阿拉伯字母 | 阿拉伯语、波斯语等 | 右到左、字母连写 | 长短元音、变体字形 |
| 希伯来字母 | 希伯来语 | 右到左、元音点可选 | 辅音骨架和 niqqud |
| 婆罗米系文字 | 印地语、泰语等 | 元音符号、组合字符 | 音节结构和附加符号顺序 |
| CJK | 中文、日语、韩语 | IME 候选、汉字变体 | 字形地区差异和字体选择 |

## Unicode 与规范化

很多带重音字符有两种写法：

- 预组合字符：`é` 是一个字符。
- 分解序列：`e` 加上组合重音。

视觉上可能一样，但搜索、排序和文件名比较时可能不同。笔记里优先直接输入预组合字符；处理脚本、文件名和批量文本时保持 UTF-8，并尽量避免混合不同规范化形式。

## 字体选择

- CJK 内容优先选带地区变体的字体，例如 SC、TC、J、K。
- 阿拉伯文需要支持连写和变音符定位的字体。
- 天城文、泰文等复杂文字需要现代 shaping 引擎支持。
- 学习资料截图最好注明字体，因为字形差异会影响辨认。

## 文件和网页习惯

- Markdown 文件保存为 UTF-8。
- 文件名可以用本地文字，但跨平台同步时要注意大小写和规范化差异。
- 网页优先声明 `lang`，例如 `zh-Hans`、`ja`、`ar`。
- 搜索资料时同时尝试原文、拉丁化和英文名。
